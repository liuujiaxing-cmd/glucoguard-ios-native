# 公益 App 矩阵官网规划 (Website Plan)

## 1. 网站定位
*   **名称**: OpenHealth (暂定名) / 开放健康公益计划
*   **Slogan**: "用代码温暖生命，让科技惠及每一个人"
*   **风格**: 极简、温暖、信任感（主色调：医疗蓝 + 温暖橙）

## 2. 站点结构图 (Sitemap)

### 2.1 首页 (Home)
*   **Hero Section (首屏)**:
    *   大标题：我们为慢性病患者开发免费、好用的工具。
    *   背景图：生活化的场景（老人用手机、家庭温馨画面）。
    *   主按钮：[浏览我们的 App] [支持我们]
*   **我们的使命 (Mission)**: 解释为什么要由个人开发者/志愿者来做这件事（打破商业软件的昂贵壁垒，保护隐私）。
*   **项目展示 (Featured Apps)**:
    *   **SugarGuard**: 您的智能血糖管家 (已上线)。
    *   **Project B (待定)**: 下一个公益计划预告。

### 2.2 App 详情页 (Product Page - SugarGuard)
*   **功能亮点**: 智能记录、AI 建议、无广告、离线可用。
*   **下载链接**: App Store 徽章，APK 下载链接。
*   **用户评价**: 真实的患者反馈。
*   **开源地址**: GitHub 链接（增加信任度）。

### 2.3 博客/故事 (Blog/Stories)
*   记录开发背后的故事。
*   分享用户的康复日记。
*   健康科普文章（可由 AI 辅助生成）。

### 2.4 透明度报告 (Transparency)
*   **资金流向**:
    *   服务器成本: ¥XXX/月
    *   域名费用: ¥XXX/年
    *   广告收入: ¥XXX/月 (如有)
*   **承诺**: "所有收入仅用于覆盖运营成本，结余将捐赠给红十字会/相关基金会。"

## 3. 技术选型建议

如果你想快速上线且易于维护，推荐以下方案：

### 方案 A：Framer (设计感强，零代码)
*   **优点**: 像画图一样做网站，动效极其精美，非常适合展示 App。
*   **缺点**: 自定义域名可能需要付费。

### 方案 B：GitHub Pages + Astro/Hugo (完全免费，程序员首选)
*   **优点**: 托管免费，绑定域名免费，速度快，不仅是官网也是技术博客。
*   **实现**: 找一个好看的 "App Landing Page" 主题，改改文字和图片推送到 GitHub 即可。

### 方案 C：Notion + Super (最快内容管理)
*   **优点**: 你只需要在 Notion 里写文档，网站自动生成。更新内容就像写笔记一样简单。

4.  **上线首页**: 先做一个简单的单页 (Landing Page)，放上 SugarGuard 的下载链接。
